도커(Docker)와 쿠버네티스(Kubernetes) 본질적으로는 컨테이너와 오케스트레이션을 가리키는 말

도커: 한개의 컨테이너를 관리, 미지를 컨테이너에 띄우고 실행하는 기술
- 컨테이너 : 마이크로서비스로 사용됨(큰 어플을 서비스 단위로 잘라 빠르게 배포하는 것)
- 각각 분리되어 사용되므로 개별 변경사항이 분일된 다른 기능에 영향을 미치지 않습니다. 

* 가상머신 vs 컨테이너

- 가상머신
Server > Host OS > Hyperisor > 각각의 Guest OS가 설치된 VM 구동
장점
- Host OS가 window여도 Guest OS 로 Linux 를 사용할 수 있습니다.
- 보안적으로 Host OS, Guest OS가 분리되어 있어 각각의 VM에 피해가 가지 않습니다.
단점
- VM 마다 무거운 Guest OS를 띄우기 때문에 컨테이너에 비해 속도가 느립니다.

- 컨테이너
Server > Host OS > Docker Engine > 컨테이너 구동
장점
- 하나의 Host OS 를 공유하기 때문에 컨테이너 별로 무거운 OS를 띄우지 않아 컨테이너의 속도가 훨씬 빠릅니다. 
단점
- Host OS가 window여도 Guest OS 로 Linux 를 사용할 수 없습니다.
- 보안적으로 컨테이너가 뚫리면 다른 컨테이너와 Host OS 가 위험해질 수 있습니디.

컨테이너는 독립적이고, 동적입니다.
- 트래픽에 따라 jave 컨테이너의 수를 늘리고 줄일 수 있습니다.
- 즉, docker 덕분에, 매번 새로운 서비스를 만들 때마다 새로운 서비스를 설정할 필요가 없는 것입니다.
- 또한, 하나의 서버에서 각기 다른 환경의 컨테이너를 설정할 수 있고, 이 컨테이너들은 각각 분리, 독립되어 있기 때문에 더욱 효율적입니다.


쿠버네티스 : 컨테이너 오케스트레이션 툴, 도커를 관리하는 툴, ex) 0월 0일 0시에 100개의 컨테니어를 자동으로 생성해야지
* 오케스트레이션이란
- 컨테이너 숫자가 많아지면 관리와 운영이 어려워지니까 다수의 컨테이너 실행을 관리하고 조율하는 시스템입니다.
- 컨테이너의 생성과 소멸, 시작, 중단 시점 제어, 로드 밸런싱, 클러스터링 등 
- 다른 오케스트레이션 툴은 도커 스윔, ECS, Nomad 등이 있다.

쿠버네이스의 특징
- 자동화된 복구
- 로드 밸런싱: 웹사이트의 니즈에 맞게 컨테이너 숫자 자동으로 조절
- 무중단 : 점진적인 업데이트를 제공하기 때문에 서비스 일시 중단 필요없음
- 호환성 : 도커 컨테이너 기반의 오픈 소스이므로 클라우드 환경을 이전할 수 있다.
